# AGENTS.md

Guidance for agents working in this Nix flake.

## Build And Test

- `nix flake check` validates flake outputs when full validation is needed.
- `nix eval .#nixosConfigurations.home-stockholm.config.system.build.toplevel.drvPath` is a quick NixOS evaluation check.
- `./nix-build.sh [hostname]` builds and switches the detected host.
- `./nix-rollback.sh [hostname]` rolls back the detected host.

## Current Architecture

- `flake.nix` defines Darwin hosts and NixOS hosts.
- `hosts/home-stockholm/` is the homeserver NixOS configuration.
- Workloads run in Kubernetes; Helm charts and Helmfile live in the sibling `k8s/` tree.
- Docker remains enabled as a daemon, but Nix no longer defines Docker Compose service stacks.
- `hosts/home-stockholm/services.d/` is backup metadata only.
- `hosts/home-stockholm/backup/` owns Restic config, scheduled backup units, and the `svc` backup CLI.
- `vault-agent.nix` renders host-side secrets needed by backups and host services.

## Homeserver Services

`services.d/` entries should stay small. A service normally contains:

```nix
{
  name = "service-name";
  backup = {
    enable = true;
    paths = [];
    kubernetes = {
      namespace = "namespace";
      deployments = ["deployment"];
      pvcs = ["pvc-name"];
    };
    tags = ["service-name"];
    exclude = [];
    policy = { daily = 10; weekly = 4; monthly = 4; };
  };
}
```

Use raw `paths` for host filesystem backups and `kubernetes.pvcs` for Kubernetes PVC backups. Do not add Compose metadata, deployed files, or service-specific Docker secrets here.

## Backup CLI

The `svc` tool is backup/restore only:

```bash
svc list [--backup-env local|remote]
svc list-backups <local|remote> <service>
sudo svc backup <local|remote> <service|all>
sudo svc restore <local|remote> <service> [snapshot|latest] [--verify-includes]
```

`svc` reads `/etc/svc/services.json`, generated by Nix from `services.d/`. Kubernetes PVC backups resolve the current local-path PVC backing directories and store metadata needed for later restores.

## Conventions

- Reference packages as `${pkgs.name}/bin/tool` in Nix-generated scripts.
- Keep secrets in Vault templates via `homeserver.vault.secrets`; never hardcode credentials.
- Avoid adding host-level service management to `svc`; Kubernetes deployment state belongs in Helm/Kubernetes config.
- Prefer small service metadata modules over cross-cutting helper abstractions.
- Do not modify `hardware-configuration.nix` unless the hardware really changed.
